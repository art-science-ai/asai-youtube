## üìù Notes for Production

### Review Video Template
Each tool review should follow this structure:

* **Hook (0:00-0:45)**: Problem statement, tool reveal, value proposition
* **Overview (0:45-2:00)**: What is it, who made it, why does it exist, key features
* **Demo (2:00-6:00)**: Real workflow demonstration with actual use case
* **Verdict (6:00-8:00)**: Pros/cons, who it's for, when to use it, rating
* **Outro (8:00+)**: Alternatives mentioned, call to action

**Consistent Elements:**
* Always show installation/setup process
* Demo at least one real-world use case
* Provide clear verdict with specific use cases
* Mention 1-2 alternatives at the end

### Thumbnail Patterns
* **Review videos**: Tool logo + 3 key features + rating/score badge
* **Comparison videos**: Split screen, red X vs green check
* **Tutorial videos**: "Before/After" with clear transformation
* **Opinion videos**: Controversial statement + question mark
* **How-to videos**: Numbered steps + result preview

### Title Formulas That Work
* **Review specific**: "[Tool Name] Review - [Value Proposition]"
* **Review specific**: "Is [Tool Name] Worth It in 2025?"
* **Review specific**: "I Tried [Tool] for [Time Period] - Here's My Verdict"
* "I [did X] with [Y] tools - Here's What Happened"
* "[Tool A] vs [Tool B] - Which One Wins?"
* "Why Everyone Is [doing X] Wrong"
* "The [Adjective] Guide to [Topic]"
* "Stop [X], Start [Y] Instead"
