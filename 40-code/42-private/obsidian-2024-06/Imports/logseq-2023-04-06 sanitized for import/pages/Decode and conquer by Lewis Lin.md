- Product design
	 - Famework - CIRCLES
		 - Comprehend the situation
			 - Context: who, what, when, where, how

			 - Goals, business objectives, metrics: revenue, market share, customer acquisition, retention, conversion, satisfaction 

			 - Constraints and assumptions: deadlines, resources, scalability, platform, geography

		 - Identity the customer (users / user personas)
			 - Demographics, behaviors, needs and goals

		 - Report the customer's needs
			 - Use cases / pain points / user stories

		 - Cut through prioritization
			 - RICE (just use Impact and Effort for short version)

		 - List solutions
			 - Methods
				 - Reversal method (reverse some workflows)

				 - Attribute method (recombine product attributes)

				 - Why method (question the status quo)

			 - Tips
				 - Think BIG!

				 - Go broad: 10+ ideas

		 - Evaluate trade-offs
			 - Critique the solutions

		 - Summarize recommendation (RRRN)
			 - Recommendation

			 - Reasons

			 - Risks

			 - Next steps

	 - Framework - Design evaluation
		 - Feature description: walk through the user journey

		 - Usefulness: what problem is it solving? for whom?

		 - User-friendliness

	 - Design criteria
		 - Usefulness: Is it solving a problem or addressing a goal?

		 - User-friendliness: Is it intuitive and easy to understand?

		 - Other criteria
			 - Useful

			 - Easy to understand

			 - Innovative

			 - Aesthetic

			 - User friendly

			 - Intuitive

			 - Solving the problem

			 - Advertising funnel: search engine terms, CPA, CPC

		 - Engagement and retention
			 - Active users: daily / weekly / monthly

			 - Sessions: number / duration / time spent

		 - Monetization
			 - CLTV

			 - ARPU

			 - MRR

			 - Churn rate

		 - Other
			 - Uptime

			 - Failure rate

			 - NPS

	 - Prioritize metrics
