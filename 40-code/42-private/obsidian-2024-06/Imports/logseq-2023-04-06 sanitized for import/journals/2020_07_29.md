- [[MCIT-593]] Module 13: Allocating Memory
id:: 7d63cd86-3d28-40c9-bee9-219dc24fdec0
	 - Dynamic memory
		 - malloc / free
			 - malloc and free interact with dynamic memory, i.e., the heap

			 - Calling malloc will allocate heap memory and return a pointer to the heap memory address. Calling free will cancel the allocation / reservation so that heap memory can be re-allocated later.

			 - When using nested mallocs, we need to free in the reverse order!

		 - How long does it last? Global memory vs. heap memory vs. stack memory
			 - Stack memory lasts until another function takes over the stack

			 - Heap memory lasts until it's freed

			 - Global / static memory lasts for the duration of the program

		 - Heap vs. stack
			 - Putting data on stack requires you to specify size in advance. Heap can be dynamically allocated.

	 - Linked lists
		 - Link data structures together using pointers so that we allocate dynamically (e.g. I don't need to know in advance how many customer records I need).

		 - How to program a linked list?
			 - Define the struct in the header file.

			 - Add node function: traverse the list using a temp pointer to look for the end node and add there.
				 - 1. First, Add new customer without linking: Malloc space for new customer, Add id, Malloc space for customer name and copy name in, set next to null

				 - 2. Then, link it to list: create head pointer; update list pointer to point to the end; then point list pointer to the new customer 

			 - Search node function: traverse list until the item matches your search (e.g. id == id)

			 - Delete node function: use two temp pointers - one to search, and one to connect previous to next (remember to free memory!)

			 - (videos includes programming implementation details)

	 - Dynamic memory pitfalls
		 - Memory leaks: forgetting to call free. Java does this "garbage collection" automatically, bu C doesn't.

		 - Buffer overflow: Writing more memory than allocated
